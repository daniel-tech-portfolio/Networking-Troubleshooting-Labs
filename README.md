# Networking Troubleshooting Labs

This repository contains hands-on networking troubleshooting scenarios designed to demonstrate practical diagnostic skills used in IT support, help desk, and junior network administration roles.

Each lab simulates a common networking issue and documents the troubleshooting process used to identify the cause, test possible fixes, and verify resolution.

---

## Skills Demonstrated

- TCP/IP troubleshooting
- DNS diagnostics
- Default gateway analysis
- Network latency investigation
- Command-line troubleshooting
- Problem isolation
- Technical documentation

---

## Tools Used

The following tools and commands are used throughout these labs:

- `ipconfig`
- `ping`
- `tracert`
- `nslookup`
- `netstat`
- `arp`
- `pathping`
- Wireshark (optional)

---

## Labs Included

### Lab 1 – No Network Connectivity
A workstation cannot access the internet or internal resources. This lab focuses on identifying DHCP and IP configuration issues.

### Lab 2 – DNS Resolution Failure
A workstation can reach external IP addresses but cannot browse by hostname. This lab focuses on DNS server validation and name resolution testing.

### Lab 3 – Gateway Misconfiguration
A workstation can reach local devices but cannot reach outside networks. This lab focuses on identifying and correcting an incorrect default gateway.

### Lab 4 – Slow Network Diagnosis
A user reports slow performance when accessing shared resources and websites. This lab focuses on packet loss, route tracing, and network statistics.

---

## Troubleshooting Method

Each lab follows a structured approach:

1. Identify the issue
2. Gather information
3. Test likely causes
4. Isolate the fault
5. Apply a fix
6. Verify the fix
7. Document the result
